Upload Base Capital CSV

Capital CSV Specs ⬇️ Download Template

Required CSV Format
Column Type Description
Broker String Broker name (e.g. IBKR, MOOMOO).
Date Date Date in YYYY-MM-DD or YYYY-MMM DD format.
Remarks String Transaction description or remarks.
Amount Decimal Net capital injection amount.
Key Rules & Assumptions
  • Four Columns Required: The uploaded CSV must contain exactly 4 columns in the order: Broker, Date, Remarks, and Amount.
  • Full Overwrite: Seeding a new capital CSV will fully overwrite and delete all existing capital entries in the database.
  • Case Insensitive: Broker names are stored as uppercase (e.g. IBKR, MOOMOO).
Import IBKR Dividends (XML)

IBKR XML Specs

Export Documentation
  • Refer to the IBKR Dividend Import Guide for details on the columns expected in the XML which is exported via IBKR Flex Queries.
  • JSON import is not natively available via the GUI since it is a customized JSON format of the XML (meant for advanced users anyway). XML is the native format exported by IBKR.
Expected XML Structure
  • Must be generated as a Flex Query Activity Report from the Interactive Brokers Account Management portal.
  • The parser extracts <CashTransaction> tags where the attribute type is set to Dividends or Payment In Lieu Of Dividends.
Fallback Portfolio Logic
  • Symbol Mapping: If the dividend's ticker symbol is already held in an active portfolio, it is automatically routed to that portfolio. If the symbol is held across multiple IBKR portfolios, it is mapped to the portfolio containing the most recent transaction for that symbol.
  • Fallback Assignment: If the ticker is new and has no transaction history in the system, the dividend is assigned to the selected Fallback Target Portfolio.

Export Documentation
  • These files should be generated directly from the website following the step-by-step instructions in the Data Migration Guide.
Expected Holdings Headers
Holding, Holdings' name, Note, Shares, Currency, Share price, Country, and Portfolios
Expected Transactions Headers
Event, Date, Symbol, Price, Quantity, Currency, FeeTax, Exchange, and Note
Key Rules & Assumptions
  • Portfolio Filter: If the Portfolios column in Holdings lists multiple items, the row is skipped unless it matches your targeted portfolio.
  • Duplicate Safeguard: Transactions matching on execution timestamp, price, and ticker symbol are skipped to avoid database duplicates.
